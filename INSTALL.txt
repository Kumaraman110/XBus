AgenTel — Windows release asset
===============================

AgenTel is a durable local coordination layer that lets independently-launched
Claude Code CLI sessions on ONE computer, under ONE OS user, discover each other
and exchange messages. It is an independent, unofficial open-source project by
Aman Kumar; it is not affiliated with or endorsed by Anthropic.

This file is the self-contained install guide for the RELEASE ASSET (this ZIP).
You do NOT need a source checkout, a git clone, "npm install", or any network
access to install from this asset. Everything needed is already inside this
folder.


Requirements
------------
  - Windows.
  - NOTHING else. This official release ZIP includes its OWN pinned Node runtime
    (runtime\node.exe). You do NOT need to install Node, npm, or NVM; you do NOT
    need admin rights; and you do NOT need to change PATH. A Node already on your
    PATH (even an unsupported one like Node 25) is IGNORED — the installer always
    uses the bundled, integrity-verified runtime.
  - (Source checkouts only, not this ZIP: a complete supported Node 22.13+ LTS or
    Node 24 on PATH is used as a development fallback. Node 25+ is not yet
    validated.)


Install (bundled runtime; PATH-free)
------------------------------------
From a PowerShell window opened IN this extracted folder:

    .\install.ps1

That is the whole install. The installer VERIFIES the bundled runtime's SHA-256
against this asset's SHA256SUMS, then uses that runtime (by absolute path) to run
the bundled CLI's `install` from this folder. It is reversible and it does NOT
modify PATH, the registry, or your PowerShell profile. It prints the runtime
source (bundled or PATH fallback), the runtime version, and the action.

To preview first without writing anything:

    .\install.ps1 -DryRun

(If runtime\node.exe were ever missing — e.g. a source checkout, not this ZIP —
the installer would fall back to a complete supported Node 22/24 on PATH, and
the equivalent direct commands would be:)

    node .\dist\cli\main.js install --dry-run    # preview, writes nothing
    node .\dist\cli\main.js install              # user-scope install


What install does
-----------------
  1. Copies the AgenTel plugin to a user-scope install root
     (default: %USERPROFILE%\.claude\xbus-install\plugin — the on-disk path keeps
     its `xbus-install` name for upgrade compatibility with existing installs).
  2. Registers the AgenTel MCP server + lifecycle hooks for Claude Code (user
     scope): a SessionStart hook (so every session auto-registers and appears in
     the dashboard) plus the UserPromptSubmit/Stop checkpoint-delivery hooks.
  3. Creates an ACL-restricted per-user data directory with a per-install secret
     (default: %USERPROFILE%\.claude\xbus-install\data).

It writes ONLY under your user scope. It does NOT change PATH, the registry, or
any shell profile. There is intentionally no global `agentel` command in this
preview — you invoke AgenTel with `node <path>` as shown below.


Verify the install
-------------------
    node .\dist\cli\main.js doctor

`doctor` reports the installed build identity (productVersion, buildId,
compatibilityId, installedArtifactManifestSha256) and broker health.


Launch Claude Code with AgenTel enabled
---------------------------------------
    node .\dist\launcher\xclaude.js

`xclaude` (the launcher's kept alias) spawns your own `claude` with
`--plugin-dir <installed plugin>` and
forwards your arguments verbatim. If `claude` is not on your PATH, set
CLAUDE_CODE_EXECPATH to its full path first, e.g.:

    $env:CLAUDE_CODE_EXECPATH = "C:\path\to\claude.cmd"
    node .\dist\launcher\xclaude.js

On Amazon Bedrock, delivery is at the receiver's next checkpoint (its next
prompt); idle sessions are not woken. See the project README for the honest
delivery-semantics notes.


The session dashboard (beta.5)
------------------------------
After install, just run Claude Code normally. Each session auto-registers at
SessionStart, the broker auto-starts, and an authenticated dashboard opens in
your default browser showing every session's state, readiness, last sent/
received, delivery breakdown, and audit-ledger health. To reopen/focus it, or to
get its URL without opening a browser:

    node .\dist\cli\main.js dashboard            # open/focus in your browser
    node .\dist\cli\main.js dashboard --no-open  # print a one-time open link

The dashboard is loopback-only (127.0.0.1) and every request is authenticated.


Invoke an installed AgenTel by absolute path
--------------------------------------------
After install you can run the installed copy directly (no need to keep this
extracted folder):

    node "$HOME\.claude\xbus-install\plugin\dist\cli\main.js" doctor
    node "$HOME\.claude\xbus-install\plugin\dist\launcher\xclaude.js"


Uninstall (reversible)
----------------------
    node .\dist\cli\main.js uninstall --dry-run    # preview what would be removed
    node .\dist\cli\main.js uninstall              # remove plugin + manifest; KEEPS data

By default your broker data dir + secret are RETAINED. To remove those too:

    node .\dist\cli\main.js uninstall --remove-data

Uninstall removes the plugin payload + install manifest + the MCP/hook
registration. It does NOT need to undo any PATH or shell-profile change, because
install never made one.


Integrity
---------
SHA256SUMS lists a SHA-256 for every payload file in this asset, including the
bundled runtime (runtime/node.exe). .\install.ps1 automatically verifies the
bundled runtime against SHA256SUMS BEFORE executing it, and fails closed
(refusing to run) if the entry is missing or the bytes do not match — it never
silently falls back to a global Node when a bundled runtime is present but fails
verification. To verify any file yourself on Windows:

    Get-FileHash .\runtime\node.exe -Algorithm SHA256
    Get-FileHash .\dist\cli\main.js -Algorithm SHA256

provenance.json records the exact, deterministic build identity (productVersion,
buildId, sourceCommit, compatibilityId) that `doctor` reports at runtime.
